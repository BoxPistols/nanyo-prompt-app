## はじめに ── ある日、自治体が764件のAIプロンプトをGitHubに公開した

> **要約:** 山形県南陽市が前例のない規模でAIプロンプトをオープンデータ化。その「使いづらさ」を解決するWebアプリを個人開発した。

山形県南陽市──人口約3万人のこの町が、**764件以上の生成AIプロンプト**をCC BY 4.0ライセンスでGitHubに公開しています。

その名も**「一発OK!! 市民も使える！生成AI活用実例集」**。

南陽市DX推進主幹・佐野毅氏（[@ichigonme](https://x.com/ichigonme)）が主導するこの取り組みは、日本の自治体DXとして前例のないスケールです。文章作成、業務改善、アイデア創出、プログラミングなど**9つのカテゴリ**にわたるプロンプトが、誰でも自由に使える形で公開されています。

しかし、公開されたデータには課題がありました。

- プロンプトがHTMLページに散在しており、**目的のプロンプトを探すのが難しい**
- コピーして使うまでの手順が多い
- どんなプロンプトがあるか**全体を俯瞰できない**

「せっかくの宝の山が、活用しきれていない」── この課題を解決するために、Webアプリ**「南陽市DX Prompts」**を開発しました。

**https://nanyo-prompt.vercel.app/**

南陽市の確認のもと、個人が開発・運営しているアプリです。本記事では、このアプリの機能紹介から技術的な設計判断まで、開発の全記録をお伝えします。

---

## アプリの全体像 ── 764件のプロンプトを「見つけて」「すぐ使える」

> **要約:** スマート検索、AI連携、カスタムプロンプトなど、アプリの主要機能を紹介。

### 検索：4つのモードで「見つからない」を解消

このアプリの核心は**検索機能**です。764件以上のプロンプトから、あなたが今必要としているものを瞬時に見つけ出します。

| 検索モード | 特徴 | 使いどころ |
|:---|:---|:---|
| **キーワード検索** | タイトル・カテゴリへの厳密なANDマッチ | 探したいものが明確なとき |
| **意図検索** | 同義語・関連語を自動展開して全文検索 | 「こんな感じの」で探したいとき |
| **あいまい検索** | タイプミスや表記ゆれに対応（N-gram類似度） | うろ覚えのキーワードで探すとき |
| **スマート検索**（デフォルト） | 上記3つを統合した最適な検索 | 迷ったらこれ |

例えば、検索窓に「メール」と入力するだけで、メール文面作成に関連するプロンプトが即座に表示されます。「mail」でも「めーる」でも同じ結果が得られるのは、日本語正規化処理のおかげです。

### ワンクリックでAIツールへ

見つけたプロンプトは、**ワンクリックでChatGPT・Gemini・Claudeに送信**できます。

1. プロンプトを選択
2. 変数がある場合はフォームに入力（例：「宛先」「用件」など）
3. 「コピーしてAIで使う」をクリック

これだけで、お好みのAIツールが新しいタブで開き、プロンプトがクリップボードにコピーされた状態で使い始められます。**生成AIに触れたことがない方でも、30秒で最初のAI体験ができる**設計です。

### カスタムプロンプトの作成

南陽市のプロンプトだけでなく、**自分だけのプロンプトを作成・管理**することもできます。

- **シンプル入力**: テキストエリアに自由に記述
- **セクション入力**: 目的・役割・前提条件・実行指示・ルール・出力形式・補足・変数設定の8つのフィールドに分けて構造的に記述
- **テンプレート**: 文章要約、コードレビュー、メール作成などのテンプレートから選んでカスタマイズ

変数には `{変数名}` の構文を使えば、実行時に入力フォームが自動生成されます。

### その他の機能

- **カテゴリフィルタ**: 9つの大カテゴリ（文章作成・要約、業務改善、アイデア創出・企画、情報収集・分析など）で絞り込み
- **お気に入り**: よく使うプロンプトをブックマーク
- **エクスポート/インポート**: カスタムプロンプトとお気に入りをJSONで書き出し・読み込み（追加/マージ/上書きの3モード）
- **ダークモード**: 目に優しい表示切り替え（ショートカット `D` キー）
- **キーボードショートカット**: `Cmd/Ctrl+K` で検索にフォーカス、矢印キーでカード移動など
- **オンボーディングツアー**: 初回利用時にステップバイステップで使い方を案内

## プライバシーファーストの設計

> **要約:** サーバーなし、全データはブラウザ内。ユーザーデータを一切外部送信しない。

このアプリには**バックエンドサーバーがありません**。すべてのデータはブラウザの `localStorage` に保存されます。

- お気に入り情報
- カスタムプロンプト
- テーマ設定
- オンボーディングの進捗

これらはすべてあなたのブラウザ内にのみ存在し、**外部に送信されることは一切ありません**。

データの移行が必要な場合は、エクスポート機能でJSONファイルとしてダウンロードし、別のブラウザやデバイスでインポートできます。

## 技術的な見どころ ── 日本語検索を"ちゃんと"動かす

> **要約:** 4つの検索モードの設計思想と、日本語特有の課題への対処法を解説。

ここからは、エンジニア向けに技術的な設計判断を掘り下げます。

### 日本語テキストの正規化

日本語検索で最初にぶつかる壁が**表記ゆれ**です。ユーザーが「メール」と入力しても、データ側は「めーる」「MAIL」「ＡＩ」「ａｉ」など様々な表記で書かれています。

本アプリの `normalize()` 関数は3つの正規化を行います:

```js
// 1. 全角英数字 → 半角（Unicodeオフセット 0xFEE0）
// 2. カタカナ → ひらがな（Unicodeオフセット 0x60）
// 3. 大文字 → 小文字
```

このシンプルな処理だけで、日本語検索の精度が劇的に向上します。

### スコアリングアーキテクチャ

検索結果の並び順を決めるスコアリングには、**フィールド別の重み付け**を採用しています。

```
title: 10  ← タイトル一致を最優先
c1:     5  ← 大カテゴリ
c2:     4  ← 中カテゴリ
c3:     3  ← 小カテゴリ
sub:    2  ← サブカテゴリ
tag:    2  ← タグ
id:     1  ← ID
content: 1  ← 本文
```

さらに、**完全一致には3倍のボーナス**、**全トークンが含まれる場合は1.5倍のボーナス**が加算されます。これにより、「だいたい合っている結果」ではなく「まさにこれが欲しかった結果」が上位に表示されます。

### 意図検索と同義語展開

意図検索（Intent Search）では、100以上のエントリを持つ**同義語マップ（`INTENT_MAP`）**を使って検索語を自動展開します。

```js
// 例："メール" で検索すると…
"メール作成": ["メール", "mail", "eメール", "電子メール", "メッセージ",
               "送信", "返信", "件名", "本文", "ビジネスメール", ...]
```

「メール」と入力しただけで、「ビジネスメール」「返信」「件名」といった関連プロンプトも検索結果に含まれます。逆引きマップ（`REVERSE_INTENT_MAP`）も自動生成されるため、双方向の関連語検索が可能です。

### あいまい検索のN-gram類似度

あいまい検索では**バイグラム（2文字組）のDice係数**を使った類似度計算を行います。

```
"プレゼン" → ["プレ", "レゼ", "ゼン"]
"プレゼンテーション" → ["プレ", "レゼ", "ゼン", "ンテ", "テー", "ーシ", "ショ", "ョン"]

共通バイグラム: 3 / 全バイグラム: 11
Dice係数: 2 * 3 / (3 + 8) = 0.545
```

さらに**スライディングウィンドウ方式の部分一致**も実装しており、長いテキストの一部分に対するあいまい検索にも対応しています。

### スマート検索の統合ロジック

デフォルトのスマート検索は、3つのモードを**異なる重み**で統合します:

- キーワード検索スコア: **1.0倍**（基本）
- 意図検索スコア: **0.8倍**（補助）
- あいまい検索スコア: **0.6倍**（補完）

この重み付けにより、正確なキーワード一致が常に優先されつつ、関連語やタイプミスにも対応できるバランスの良い検索結果が得られます。

## データの自動同期 ── GitHub Actions × Pythonスクレイパー

> **要約:** 毎朝自動で最新データを取得。SHA比較による差分検出で効率的に同期。

南陽市のプロンプトデータは日々更新される可能性があります。手動で同期するのは現実的ではないため、**GitHub Actionsによる自動同期パイプライン**を構築しました。

### 同期の流れ

```
毎朝 9:00 (JST)
  ↓
GitHub API でnanyo-line/promptのファイルツリーを取得
  ↓
SHAハッシュを前回と比較 → 変更ファイルのみ検出
  ↓
変更があったHTMLページをスクレイピング
  ↓
データを更新して自動コミット
```

SHA比較による**差分検出**がポイントです。764件全てを毎回スクレイピングするのではなく、変更があったファイルだけを処理するため、効率的かつAPI制限にも優しい設計になっています。

### クライアント側のデータマージ

アプリ側でも、データ更新時にユーザーのカスタムプロンプトを消さないための**マージロジック**（`syncLogic.js`）を実装しています。

- ソースプロンプト（`isUser: false`）は最新データで上書き
- ユーザー作成プロンプト（`isUser: true`）はそのまま保持
- 削除されたプロンプトはお気に入りからも自動クリーンアップ

## 技術スタック

> **要約:** React 19 + Vite、UIライブラリ不使用の軽量構成。

| 技術 | バージョン/詳細 |
|:---|:---|
| **React** | 19.2 |
| **Vite** | 7.3 |
| **UIライブラリ** | 不使用（CSS変数 + カスタムCSS） |

| **Lint** | ESLint 9 |

あえてUIライブラリに依存しない選択をしました。バンドルサイズの削減と、カテゴリ別カラーシステム（9カテゴリ × ライト/ダークの18パターン）やグラスモーフィズム効果など、アプリ固有のデザインを柔軟に実現するためです。

## 「市民も使える」の本当の意味 ── DXは技術ではなく、届けること

> **要約:** エンジニアが作ったものを、エンジニアでない人が使える形にすることがDXの本質。

南陽市のプロンプト集のタイトルは**「市民も使える！」**です。この「も」という一文字に、自治体DXの本質が詰まっています。

技術的に優れたプロダクトでも、使い方がわからなければ意味がありません。このアプリでは以下の工夫で「届ける」ことを重視しました。

- **オンボーディングツアー**: 初回アクセス時にステップバイステップで使い方を案内
- **モーダル内ツアー**: プロンプト実行画面にも個別のガイドを用意
- **レスポンシブ対応**: スマートフォンでも快適に操作可能
- **視覚的なカテゴリ分類**: 色分けにより、文字を読まなくてもカテゴリが判別可能
- **変数入力フォーム**: プロンプト内の変数が自動的に入力欄として展開

生成AIに触れたことがない市民の方が、検索窓に「メール」と打ち、プロンプトを選び、宛先と用件を入力し、ボタンを押す── **この30秒の体験**が、生成AI活用の第一歩になります。

## オープンデータ × コミュニティの可能性

> **要約:** CC BY 4.0のプロンプト + MITライセンスのアプリで、他自治体や組織にも展開可能。

南陽市のプロンプトデータは**CC BY 4.0**、このアプリのソースコードは**MITライセンス**で公開されています。

これは何を意味するか:

- **他の自治体**が同じ仕組みを自分たちの地域に導入できる
- **企業や教育機関**がプロンプト集を参考に、組織内のAI活用を促進できる
- **個人開発者**がフォークして機能を追加したり、別のデータソースに接続できる

南陽市のモデルが他の自治体に広がれば、日本全体の生成AI活用スキルが底上げされる可能性があります。**1つの町の取り組みが、全国のDXを加速させるかもしれない** ── そんな未来を感じさせるプロジェクトです。
